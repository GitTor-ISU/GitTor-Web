The format mirrors .gitconfig on purpose, so it should feel familiar: 

{{ configFileExample }}
  • port — the port the seeder listens on. Public port-forwarding isn't required, but it makes seeding much more effective for other users.
  • api_url — which GitTor web instance the CLI talks to. Point this at your own deployment if you self-host.
  • tracker1... trackerN — public BitTorrent trackers used when creating new torrents. Anywhere from one to a hundred is fine; the example list is what the GitTor team used during development.

After editing .gittorconfig, restart the seeder service: gittor service restart.
