1. gittor init project creates a new repository.
  2. Make changes and commit them with the usual git add and git commit.
  3. gittor seed publishes the repo. On first seed, you'll be prompted for a name and description. The output gives you a repository ID, torrent file, and magnet link.
  4. Share the repository ID with collaborators. They run gittor leech <repo-id> project to clone it.
  5. Inside an already-leeched repo, gittor leech with no arguments pulls the latest state.

When you seed a repo, you get three different identifiers. Each has trade-offs:

  • Repository ID — short, stable across updates, but requires the web app to resolve.
  • Magnet link — works without the web app, but changes every time the repo is updated.
  • Torrent file — same as magnet link, but a file instead of a string.

Day to day, use repository IDs. Fall back to magnet links if the web app is unreachable.
